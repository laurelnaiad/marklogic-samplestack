mlUtil

  • - service in module _marklogic

Provides utility objects and methods.

Methods

  • method:stripZone(dateish);

    Parameters

    Param Type Details
    dateish object

    A JavaScript or moment date object

    Returns

    string

    an ISO8601 string representing the date, with time zone information removed.

    This function will be unnecessary when time zones and date string handling is fully implemented. For now, it is used to ensure that the middle tier doesn't get ISO8601 time zone information that it cannot process.

  • method:objectify(obj);

    Converts and object to a "plain-old JavaScript object" by converting it to JSON and then back to an object. This causes the results of serialization to JSON to be found in the resulting object. For instance, dates will be serialized to strings. Additionally, non-enumerable properties are dropped in the resulting object.

    Does not modify its parameter.

    This method is used when validating mlModelBase-derived objects.

    Parameters

    Param Type Details
    obj object

    Any object

    Returns

    object

    a "plain old Javascript" version of the object

  • method:merge(targetObject, objectToMerge, remainingArgs);

    This is a deep merge. Objecgts are merged from right to left, so the last on is merged into the second to last and ultimately the first object in the parameters list is the final target. These merges occur on the actual objects, so if targets of these marges are not intended to be modified, clones should be used.

    This is similar to lodash's merge function.

    Parameters

    Param Type Details
    targetObject object

    The first object is the ultimate target of the merge(s).
    objectToMerge object

    an object to merge into the target
    remainingArgs *

    additional objects to merge

    Returns

    object

    Reference to the first object in the parameters list, which is the ultimate target of the merge(s).

Properties

  • moment

    object

    This is Moment.js. It is used to handle conversion between dates, in particular between ISO 8601 strings from JSON and Javascript Date objects.

    It is also used by mlModel elements when serializing and deserializing JSON dates for $http.

    Until time zone handling is fully implemented, this instance of moment is configured to operate entirely and execlusively in UTC mode.
